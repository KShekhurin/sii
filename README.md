# 🎯 Bet Game 

Проект реализует простую игру-аукцион и набор агентов (стратегий), которые могут в неё играть. Архитектура позволяет легко добавлять новых агентов и проводить эксперименты.

---
## 📁 Структура проекта

```
.
├── auction.py   # логика игры
├── agents.py    # агенты (стратегии)
└── README.md
```

---


## 🎮 Описание игры

- У каждого игрока: **200 очков**
    
- Игра длится **5 раундов**
    
- В каждом раунде:
    
    - генерируется скрытая стоимость предмета: **10–30**
        
    - игроки одновременно делают ставки 
        
    - выигрывает тот, у кого ставка больше
        

### Результат раунда:

| Условие   | Результат      |
| --------- | -------------- |
| Победа    | `+value - bid` |
| Поражение | `0`            |
| Ничья     | `0`            |
### Условие победы:

Побеждает тот, кто по окончании всех раундов будет иметь больше очков (ничья в случае равенства)

## ▶️ Как запустить игру

1. Убедитесь, что установлен Python 3.8+
    
2. В `auction.py` (или отдельном `main.py`) создайте агентов, параметром назначьте им имя:
     
3. Создайте игру, передав параметрами двух разных (в смысле объектов) агентов.
    
4.  Используйте метод play игры.
    
Пример:

```python
from auction import Game
from agents import AdaptiveAgent, RandomAgent

agent1 = AdaptiveAgent("Adaptive")
agent2 = RandomAgent("Random")

game = Game(agent1, agent2)
game.play()
```

Запуск:

```bash
python auction.py
```

или через отдельный файл:

```bash
python main.py
```

## 🤖 Описание реализованных агентов

---

### 🔹 `RandomAgent`

Ставит случайное значение от 0 до 30.

---

### 🔹 `DoNothingAgent`

Всегда ставит 0.

---

### 🔹 `HumanAgent`

Считывает ставку из консоли.

---

### 🔹 `MirrorAgent`

- 1-й раунд: случайная ставка
    
- далее: копирует ставку оппонента
    

---

### 🔹 `EVAgent`

- ориентируется на ожидаемую стоимость (≈20)
    
- избегает переплаты
    

---

### 🔹 `AdaptiveAgent`

- если проигрывает → увеличивает ставки
    
- если выигрывает → снижает риск
    
- в конце игры становится агрессивнее
    

---

### 🔹 `MinimaxAgent`

- перебирает возможные ставки
    
- выбирает стратегию с максимальным гарантированным результатом
    

---

### 🔹 `BayesianAgent`

- оценивает среднее и дисперсию ставок оппонента
    
- выбирает ставку с максимальным ожидаемым выигрышем
    

---

### 🔹 `PunisherAgent`

- реагирует на агрессию оппонента
    
- в остальных случаях действует случайно

## 🛠 Как создать своего агента
### 🧠 Интерфейс агента

Все агенты наследуются от базового класса `Agent`.

```python
class Agent:
    def make_bid(self, context: dict) -> int:
        pass
```

---

### 📦 Формат `context`

```python
{
    'round': int,
    'my_points': int,
    'opponent_points': int,
    'history': [
        {
            'my_bid': int,
            'opponent_bid': int,
            'value': int
        }, ...
    ]
}
```

---

Чтобы добавить нового агента:

1. Создай наследника класса `Agent`
    
2. Реализуй `make_bid`, возвращающий целое неотрицательное число

Пример:

```python
from agents import Agent

class MyAgent(Agent):
    def make_bid(self, context):
        my_points = context['my_points']
        return min(10, my_points)
```

---

### 🔧 Вспомогательные методы

### `clip_bid(bid, my_points)`

Гарантирует корректную ставку:

- не меньше 0
    
- не больше 30
    
- не больше доступных очков
    

```python
bid = self.clip_bid(bid, my_points)
```

## ⚠️ Важные ограничения

- Агент **не управляет очками**
    
- Вся логика игры находится в `Game`
    
- Агент работает только через `context`, валидацию которого обеспечивает игра
    
 - Игра обрезает ставку, превышающая максимум.  Меняет ставку на 0 при любых других неверных форматах ставки

## 🧩 Описание классов

### `class Game`

Основной класс, управляющий игрой.

#### Конструктор

```python
Game(agent1: Agent, agent2: Agent, rounds: int = 5)
```

- `agent1`, `agent2` — игроки
    
- `rounds` — количество раундов
    

---

#### `play()`

Запускает игру:

- выполняет раунды
    
- запрашивает ставки
    
- обновляет очки
    
- выводит результат
    

---

#### `_get_context(agent, opponent, round_number)`

Формирует состояние для агента.

---

#### `_validate_bid(bid, max_points)`

Проверяет корректность ставки:

- неотрицательная
    
- не больше доступных очков
    
- приводит к допустимому диапазону
    

---


### `class Agent`

Абстрактный базовый класс.

#### Методы:

- `make_bid(context)`  Делает запись ставки
 - Принимает контекст, должен вернуть целое неотрицательное
 - **обязателен для реализации**
