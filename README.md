## 1. Правила игры

Игра проходит между двумя участниками и состоит из `n` раундов.

Каждый раунд разбит на `m` подраундов. В начале раунда генерируется секретное число `secret ∈ [a, b]`, которое не знает ни один из игроков.

1. **Одновременные ставки**  
   В каждом из `m` подраундов оба игрока одновременно (не видя ставку соперника) выбирают целое число — свою ставку.  
   Ставка не может быть отрицательной и не может превышать текущий баланс игрока.

2. **Накопление ставок**  
   Все ставки (по `2·m` штук) запоминаются и становятся известны после завершения всех подраундов.

3. **Определение победителя раунда**  
   По окончании `m` подраундов находится максимальная ставка `max_bid` среди всех `2·m` ставок.  
   - Игрок, который сделал хотя бы одну ставку, равную `max_bid`, платит `max_bid` (вычитается из его баланса) и получает секретное число `secret` (прибавляется к балансу).  
   - Это происходит **один раз** для каждого такого игрока, независимо от того, сколько раз он поставил максимальную сумму.  
   - Если максимальную ставку сделали оба — каждый платит и получает выигрыш.

4. **Повторение**  
   Раунды повторяются `n` раз. Начальный баланс `s` одинаков для обоих игроков.

5. **Победа**  
   После `n` раундов побеждает игрок с большим итоговым балансом. При равенстве объявляется ничья.

**Пример:**  
`a = 10, b = 20, m = 5, s = 1000`. Средний бот ставит `15` каждый ход. В раунде секретное число `18`. Максимальная ставка `15` (у обоих). Баланс каждого: `1000 - 15 + 18 = 1003`. За 10 раундов баланс станет `1030`.

## 2. Архитектура проекта

Код построен по принципу «разделяй и подключай». Все классы делятся на базовые абстракции и конкретную реализацию аукциона.

### 2.1 Базовый каркас (Player, Game)

```python
from abc import ABC, abstractmethod
from typing import List, Dict, Any

class Player(ABC):
    @abstractmethod
    def make_decision(self, state: Dict[str, Any]) -> int:
        """Принять решение на основе состояния игры."""
        ...

# Сброс состояния перед новой игрой (опционально)
class Game(ABC):
    def __init__(self, players: List[Player]):
        self.players = players

    @abstractmethod
    def run(self) -> int:
        """Запустить игру и вернуть индекс победителя (0, 1 или -1)."""
        ...
```
Эти классы не привязаны к конкретной игре, что позволяет на их основе создавать любые пошаговые противостояния (крестики-нолики, дилемма заключённого и т.д.).

2.2 Конкретная игра AuctionGame
Хранит параметры a, b, n, m, s и реализует метод run(), в котором:

Для каждого раунда генерируется secret.

Выполняется m одновременных подраундов.

В конце раунда подводятся итоги и обновляются балансы.

Возвращается победитель.

Ключевая деталь: перед вызовом make_decision игроку передаётся словарь state с актуальной общедоступной информацией (подробнее в разделе 4).

3. Запуск готовой игры
Убедитесь, что установлен Python 3.8 или выше.

Запустите основной скрипт.

Введите параметры игры: границы секретного числа, число раундов и подраундов, начальный баланс.

Выберите тип первого и второго игрока из предложенного списка.

Наблюдайте за результатами в консоли.

Доступные боты:

random — случайная ставка от 0 до баланса.

constant — фиксированная ставка (запрашивается при выборе).

average — среднее арифметическое (a + b)/2.

lastbid — повторяет последнюю уже известную ставку соперника (0, если истории нет).

human — человек вводит ставку с клавиатуры.

4. Как создать своего бота
Вам нужно лишь унаследоваться от Player и реализовать свою стратегию.

4.1 Что содержит state?
Словарь со следующими ключами:

Ключ	Тип	Описание
round	int	Номер текущего раунда (начиная с 0)
subround	int	Номер подраунда внутри раунда (0...m - 1)
bids	list[tuple]	Все ставки, уже сделанные в прошлых подраундах этого раунда. Каждый элемент — (id_игрока, ставка). В текущем подраунде ставок ещё нет!
my_balance	int	Ваш текущий баланс
opponent_balance	int	Баланс соперника
my_id	int	Ваш идентификатор (0 или 1)
params	dict	Исходные параметры игры: a, b, n, m, initial_balance
Важно: bids внутри подраунда не содержит ставку ни вашу, ни соперника в этом подраунде. Игроки принимают решение одновременно, видя только историю предыдущих подраундов.

4.2 Пример бота
```python
class RandomBot(Player):
    """Ставит случайную сумму от 0 до b."""
    def make_decision(self, state: Dict[str, Any]) -> int:
        return random.randint(0, state['params']['b'])
```
4.3 Бот с памятью
Если боту нужно хранить историю между раундами, переопределите reset():

```python
class AdaptiveBot(Player):
    def reset(self):
        self.history = []

    def make_decision(self, state):
        # Сохраняем информацию о раунде
        self.history.append(state['bids'])
        # ... стратегия на основе всей истории
```
4.4 Регистрация в меню
Чтобы ваш бот появился в списке выбора, добавьте его в реестр BOTS_REGISTRY:

```python
BOTS_REGISTRY = {
    ...
    "my_bot": MyCleverBot,
}
```
Теперь при запуске игры можно будет набрать my_bot.

5. Запуск
Вы можете запустить игру между своим ботом и одним из встроенных прямо в коде:

```python
game = AuctionGame(
    [MyCleverBot(), AverageBot()],
    a=10, b=20, n=5, m=3, s=500
)
winner = game.run()
```
Логи раундов будут выведены в консоль. Анализируйте, как менялся баланс, и дорабатывайте стратегию.
